# Remote Desktop на Python (замена AnyDesk)

Самописный клиент удалённого доступа: просмотр экрана + полное управление
мышью и клавиатурой. Трафик шифруется end-to-end (AES-GCM, ключ из пароля),
поэтому даже промежуточный relay-сервер не видит содержимое.

## Быстрый старт — готовая программа (рекомендуется)

Самый простой путь: одно окно `app.exe` (выбираете режим, вводите адрес и
пароль, жмёте «Запустить» — командная строка не нужна).

1. Соберите exe один раз:
   ```powershell
   powershell -ExecutionPolicy Bypass -File build_exe.ps1
   ```
   Готовый файл — `dist\app.exe` (~38 МБ, Python ставить не нужно).
2. Скопируйте `app.exe` на оба ПК.
3. На удалённом ПК: режим **«Этот ПК (дать управление собой)»** → Запустить.
4. У себя: режим **«Подключиться к ПК (управлять)»** → Запустить.

Для связи между разными сетями (за NAT) выберите соединение «Через relay» и
поднимите `relay.py` на VPS (см. вариант Б ниже).

## Компоненты

| Файл | Где запускать | Назначение |
|------|---------------|------------|
| `app.py` / `app.exe` | оба ПК | GUI-лаунчер (host + client в одном окне) |
| `host.py`   | удалённый ПК (которым управляют) | захват экрана + ввод (CLI) |
| `client.py` | ваш ПК                           | окно-просмотрщик + управление (CLI) |
| `relay.py`  | VPS с публичным IP               | посредник (нужен при NAT) |

> Ниже описан запуск через командную строку (`host.py`/`client.py`) — он
> эквивалентен GUI и удобен для отладки/автозапуска.

## Установка

На обоих ПК (Python 3.9+):

```powershell
pip install -r requirements.txt
```

> На `relay` (VPS) ставить ничего не нужно — он использует только стандартную библиотеку.

---

## Вариант А. Прямое подключение (LAN / VPN / Tailscale / проброс порта)

Самый простой и быстрый. Подходит, если ПК в одной сети **или** между ними
есть туннель (корпоративный VPN, [Tailscale](https://tailscale.com) — бесплатно
и без публичного сервера).

На удалённом ПК:
```powershell
python host.py --listen 5900 --password ПРИДУМАЙТЕ_ДЛИННЫЙ_ПАРОЛЬ
```

На вашем ПК:
```powershell
python client.py --connect IP_УДАЛЁННОГО:5900 --password ТОТ_ЖЕ_ПАРОЛЬ
```

`IP_УДАЛЁННОГО` — это IP в LAN, VPN или Tailscale-адрес (`100.x.y.z`).

---

## Вариант Б. Через relay (вы за NAT, разные сети — ваш случай)

Так работают AnyDesk/TeamViewer: обе стороны подключаются **наружу** к общему
серверу, проброс портов не нужен. Понадобится дешёвый VPS с публичным IP.

1. На VPS:
   ```bash
   python3 relay.py --port 5800
   ```
   Откройте порт 5800 в фаерволе/security group.

2. На удалённом ПК:
   ```powershell
   python host.py --relay VPS_IP:5800 --id myroom --password ПАРОЛЬ
   ```

3. На вашем ПК:
   ```powershell
   python client.py --relay VPS_IP:5800 --id myroom --password ПАРОЛЬ
   ```

`--id` — произвольное имя «комнаты»; host и client с одинаковым id находят друг
друга. Пароль relay не знает, поток зашифрован между host и client.

> **Без своего VPS?** Поднимите туннель к `host.py --listen`:
> Tailscale (проще всего), Cloudflare Tunnel, ngrok или проброс порта на роутере.

## Управление

- Мышь и клавиатура транслируются на удалённый ПК, когда окно в фокусе.
- Окно можно растягивать — картинка масштабируется.

Горячие клавиши в окне клиента:

| Сочетание | Действие |
|-----------|----------|
| **Ctrl + Alt + Q** | выход из клиента |
| **Ctrl + Alt + M** | переключить монитор удалённого ПК |
| **Ctrl + Alt + S** | выбрать и отправить файл на удалённый ПК |
| **Ctrl + Alt + I** | показать/скрыть статистику канала (FPS, RTT, KB/s) |

## Дополнительные возможности

- **Буфер обмена** синхронизируется автоматически в обе стороны (текст). Скопировали
  на одном ПК — вставляете на другом.
- **Передача файлов:** Ctrl+Alt+S открывает диалог выбора файла; он сохраняется на
  удалённом ПК в папку `~\RemoteDesktop_received` (меняется флагом `--downloads` у host).
- **Несколько мониторов:** Ctrl+Alt+M циклически переключает мониторы удалённого ПК;
  текущий монитор и их число показаны в заголовке окна.

## Настройка качества/скорости

Задаётся **на стороне хоста** (ПК, который показывает экран) — в окне `app.exe`
в режиме «Этот ПК» есть поля **чёткость / FPS / масштаб**. Меняются без пересборки,
запоминаются.

- **масштаб** — самый сильный рычаг по трафику: `75%` или `50%` уменьшают объём
  данных в ~2–4 раза, поэтому папки/окна открываются заметно быстрее (картинка чуть мягче).
  Управление при этом точное — мышь мапится на реальное разрешение.
- **FPS** — плавность. На статике трафик не тратится (отправляются только изменения).
- **чёткость** — качество JPEG (меньше = меньше трафик).

Рекомендации для канала через relay с пингом ~100 мс: **масштаб 75%, FPS 20, чёткость 50**.
Если экран всё ещё «тяжёлый» — масштаб 50%.

> Главный неустранимый фактор задержки — RTT до relay. Если сервер далеко
> географически, прямое соединение через Tailscale или relay поближе помогут
> сильнее любых настроек.

Для CLI те же параметры: `host.py --scale 0.75 --fps 20 --quality 50`.
В `host.py` сверху лежат дефолты (`JPEG_QUALITY`, `TILE`, `TARGET_FPS`, `SCALE`).

## Безопасность и важные оговорки

- **Пароль = ключ шифрования.** Берите длинный случайный (20+ символов).
- Соль фиксированная, поэтому весь смысл стойкости в самом пароле.
- `host.py` отдаёт **полный контроль** над ПК — запускайте только осознанно.
- На Windows для надёжной работы ввода/захвата иногда нужны права администратора
  (особенно чтобы управлять окнами, запущенными от админа, и UAC-экранами —
  последние недоступны без службы уровня системы).
- Согласуйте использование с политиками вашей организации: если AnyDesk
  запрещён регламентом ИБ (а не лицензией), самописный аналог может попадать
  под тот же запрет. Уточните у вашего ИТ/ИБ-отдела.

## Проверка работоспособности

Безголовый end-to-end тест (поднимает relay + host и играет роль клиента,
ввод мышью/клавиатурой не трогается):

```powershell
.\.venv\Scripts\python.exe smoke_test.py
```

Должно вывести `ИТОГ: 11/11 проверок пройдено`.

## Сборка в .exe (без установки Python на удалённом ПК)

```powershell
powershell -ExecutionPolicy Bypass -File build_exe.ps1
```

Результат — `dist\host.exe` и `dist\client.exe`, самодостаточные. Удобно,
если на удалённом ПК нет Python: копируете только `host.exe`.

> Антивирусы иногда ругаются на onefile-сборки PyInstaller (ложное срабатывание) —
> при необходимости добавьте файл в исключения.

## Ограничения текущей версии

- Без передачи звука.
- Передача файлов — только в одну сторону (client → host) и по одному файлу за раз.
- Буфер обмена синхронизируется только для текста (не изображения/файлы).
- Один клиент на host одновременно.
- UAC/экран блокировки Windows не транслируются (ограничение пользовательского
  сеанса; нужен запуск как системная служба).
